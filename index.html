Extract Boundary

Extract boundary of 2D grids

Download this project as a .zip file Download this project as a tar.gz file

This is user guide of Extract Boundary program for two dimensional grids. This program can be used as ParaView plugin or a stand alone program. Users will find building and installation process and usage examples in both terminal and ParaView.

Introduction

This package has been created to facilitate ParaView users that interact with two dimensional data grids. This package is utilized as a ParaView plugin, so the end users could take advantage of it as a new Filter. The Extract Boundary filter extracts boundaries of a 2D grid. It is robust to concave geometries, unusual cell links in a boundary point and acute and narrow bay-like shapes. Furthermore, user may opt for extracting all inner boundaries, such as lakes. This filter also identifies points with zero velocity. Inward or outward normal vectors are also calculated at each point.

The program accepts various grid types such as structured or unstructured grids. Also it works well with different two dimensional cell types, for instance triangular, polygons, etc. Moreover, multiple file formats as supported. Common file formats are legacy *.vtk and binary *.vtu extensions.

Build Source

User can build the code from source. You can obtain the source code from github repository. Please install git package:

sudo apt-get install git

Then, create a build directory and download the source code inside the build directory: \begin{verbatim} $ mkdir /tmp/ExtractBoundary $ cd /tmp/ExtractBoundary $ git clone git://github.com/ameli/extract-boundary \end{verbatim} Now, build the code: \begin{verbatim} $ cmake . $ make $ sudo make install $ cd .. $ rm -r /tmp/ExtractBoundary \end{verbatim} Installation is system-wide for all users.

% -------------- % Install Binary % --------------

\section{Install Binary} You may use binary package instead of building code from source. Binary installation is available for Ubuntu operating system. Follow these commands in Ubuntu terminal: \begin{verbatim} $ sudo add-apt-repository ppa:ameli/extractboundary $ sudo apt-get update $ sudo apt-get install extractboundary \end{verbatim}

% ------- % Options % -------

\section{Options} Filter has the following options: \begin{description} \item [Search Boundary Mode] can be Outer boundary or outer and all inner boundaries. Default is on searching only outer boundaries, which is $\mathcal{O}(\sqrt{n})$. Inner and outer boundary search is in order of $\mathcal{O}(n)$.

\item
[Extract Velocity Boundary] is boolean. Default is on \texttt{true}.

\item
[Calculate Normals] is boolean. Default value is set to \texttt{true}.

\item
[Normals Outward] is boolean. Default value is set to \texttt{false}.

\end{description}

% ------------- % Program Usage % -------------

\section{Program Usage} You can use the program either in terminal or as a ParaView plugin. Here are examples to use it in terminal: \begin{verbatim} $ extractboundary /input-path/inputfile.vtk /output-path/outputfile.vtk \end{verbatim} You can also define only output filename without path. In this case, program writes output file in the same directory of input path. \begin{verbatim} $ extractboundary /input-path/inputfile.vtk outputfile.vtk \end{verbatim} You may not define any path. So it considers the current directory for both input and output path: \begin{verbatim} $ extractboundary inputfile.vtk outputfile.vtk \end{verbatim} If user do not specify output file, it writes output file with the same name as input file name with a \texttt{-boundary} suffix and the same file extension of input file. \begin{verbatim} $ extractboundary /input-path/inputfile.vtk \end{verbatim} or \begin{verbatim} $ extractboundary inputfile.vtk \end{verbatim} In this case, output file name would be \texttt{inputfile-boundary.vtk}.

For help, use \texttt{-h}, for get program information use \texttt{-i} and for see the license, use \texttt{-l} options. \begin{verbatim} $ extractboundary -h $ extractboundary -i $ extractboundary -l \end{verbatim} A sample vtk output file is similar to the following: \lstinputlisting[caption={Sample output vtk file},label=vtk,style=my-listing-style]{/home/sia/code/projvtk/Filter/ExtractBoundary/doc/OutputData.txt}
